// Session 2 : Hands On.

--Introduction.

In this section we delve deeper into using various tinkering tools to play around with things in the lecture along with learning about their importance in cultivating a spirit of tinkering among its users.

One of the most noticeable solutions turned out to be the "Makedo cardboard tools" that made me realise how it can be used for my design building block in a simple yet quite tinkerable way by simply using cardboard that is more practical to work with than 3D prints for the task at hand. It can also be disposed off and/or recycled in a sustainable way compared to 3D prints that release a ton of microplastics every time they interact with anything.

This also gave more clarity on the type of solution that can be feasible for the building blocks, in the last session, I discussed threads connecting blocks as a method, that might be more practical than using jigsaw connections as the flow of data between the cards can be better visualized using the threads. Blender's node system acted as an inspiration here.

--Physical prototyping building blocks.

At this point I decided to make yet another crucial pivot in design. I decided to focus on specifically one type of algorithm : processing of audio. Each card still behaves the same way having a function or an input or an output, but now the input is audio/SD card inputs that convert digital audio into analog and each function executes some operation on the analog signal it recieves. The flow of data occurs from the left edge of a card where every thread that comes upto it acts as an input and the right edge has all threads that have outputs.

  1. First your main observations during the tinkering sessions. Can you recognise most of the aspects of the framework in action?

  2. Check up on the concept of Affordance. What is the application to the materials used in tinkering? To what extend are the original definition (Gibson) or the HCI oriented re-definition of Norman useful?

    When we talk about the word "Affordance", we think of monetary affordance of commodities. In the realm of tinkering based on the works provided and the lecture takeaways, it is clear that "Affordance" in this context refers to what the environment offers an individual. This can refer to the various tools and the subsequent solution space that gets unlocked using these tools. The way the user decides to use these tools to explore the affordance is the key process in tinkering.
