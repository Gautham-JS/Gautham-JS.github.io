// Session 4 : Session Design.

--Introduction.

This session went further into the group design by working further into a solution that can be used in context of tinkering.

--The Idea.

For this session the powerpoint slides were presented including the 2 solutions that we selected for this course.

  1. Lego inspired obstacle course to design vehicles that can traverse challenging obstacles.

    Resit : Newly Added Section

    This idea revolves around development of a playground where a mini-engineering challenge course can be defined using a set of obstacles. The goal is to engineer a mechanic system with a kit of tools provided to traverse the obstacles.

    This challenge can also be extended by inviting the community to contribute to development of obstacle courses of varying difficulties with custom objects as well, along with that the vehicle design can also evolve with community contributions.

  2. Inspired from the personal building blocks designed for algorithms, a card deck that can be used for generating music (Unlike modifying music that was done for personal block)

    Resit : Newly Added Section

    Here we explored the possibility of using physical cards inspired by MakeDo to realize algorithms using blocks that can connect. In addition, we explored specific use-cases of the "algorithm" that is developed using the cardboard blocks. For this purpose, development of music was tought of as a possible usecase where each block can be a processing unit for a musical note.

The second idea was selected as our final idea for this course.

--Scope.

For the scope of this solution, we decided to tackle the ways to generate music using card like blocks that provide input which can be mixed with other blocks to generate a sound. This can be combined with audio playback system as an output and a set of tone generators as inputs.

--Matreials.

As for materials, we would need each block capable of generating a tone that can be tuned. Thus, a microcontroller enabled unit with a dial that can provide a certain tone.

Another component we would need is a mixer that can take in 2 or more audio signals and output one signal combining all incoming signals. This would once again need microcontrollers with multiple inputs capable of signal processing.

As a base we can just use cardboard to house these units in MakeDo style units.

--Facilitation.

Facilitation for this project can be done with people who are experts in the realm of musical instruments or people who are familiar with music production.

--Process.

Process involves allowing a set of participants play around and tinker away at the musical cards and generate sounds. The possibility to use custom input cards also opens up the possibility for users to try out different samples and tracks giving very high creative freedom.

--Seed.

The seed for this project would be creation of modules for music control.
